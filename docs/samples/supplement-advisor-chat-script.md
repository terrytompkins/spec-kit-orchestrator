# SupplementAdvisor - Sample Interview Chat Script

This document contains sample user responses for testing the AI interview chat feature in the Spec Kit Orchestrator app.

**Project**: SupplementAdvisor  
**Purpose**: Web application to help users manage health supplements with AI-powered advice

---

## User Response 1: Initial Project Description

I take a lot of health supplements (e.g. vitamins, herb-based supplements). I would like to be able to use generative AI to determine:

1. Do any of the supplements conflict with each other
2. Is there a particular time of day in which a supplement should be taken?
3. Are any of the supplements I'm taking known to have any risks or lack real evidence that they actually provide an advantage?
4. If I provide some goals that I want to use supplements to help achieve, are there some supplements that I should add to my list, or replace an existing supplement with?

To implement this functionality, I would like to create a web application (based on Streamlit and a database such as SQLite or Postgresql) which allows a user to interact with an AI-empowered chat to get advise on questions like the ones above.

## AI/User Exchange 2: (To be filled during interview)

### AI Response ###
Great, that sounds like a very useful and ambitious project! Let’s delve deeper into understanding the purpose and the target user for this application.

1. Project Purpose: You mentioned using generative AI to assess supplement interactions and recommendations. Could you elaborate on why this is important? What specific problems or pain points do you think users are facing that this project aims to solve?

### User Response ###
It is difficult to track efficacy (new studies come out all the time) of supplements. AI would be able to look up current, reliable content and provide direct answers to the user. AI could also help determine from online supplement documentation whether or not there might be conflicts with various combinations of supplements the user has indicated he or she is taking. Since time of day for taking a supplement can also be a factor in determining its usefulness, that could also be researched in the AI conversation. Ideally, after the user supplies all information about current supplements, the AI in the application's chat would be able to provide a recommendation report suggesting any changes that should be made to supplements, schedule for taking them, or gaps in supplements provided goals the user may have health-wise.

### AI Response ###
That makes a lot of sense. Addressing the challenge of staying informed about the latest research and understanding supplement interactions can indeed be overwhelming for many users. Let's now focus on who the target user is.

2. Target User: Can you describe who the primary users of this application would be? Are they health enthusiasts, individuals with specific health conditions, or perhaps healthcare professionals? What are their typical characteristics, and how familiar are they with using AI-driven tools?


### User Response ###
The typical user i'm envisioning would be a health enthusiast - although if there are concerns around side-effects or risks of specific supplements, that could be called out too. it should be assumed that the user is comfortable participating in an AI-driven chat.
